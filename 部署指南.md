# DocFlow 部署指南

## 你的操作清单

整个部署过程你需要操作 **6个步骤**，预计耗时 **15分钟**。

---

## 第一步：创建GitHub仓库

### 操作
1. 打开 https://github.com
2. 点击右上角 **+** → **New repository**
3. 填写：
   - Repository name: `docflow`
   - Description: `让AI生成专业级中文PDF`
   - 选择 **Private**（或者Public都可以）
4. 点击 **Create repository**

### 等待
记住页面上显示的仓库地址，类似：`https://github.com/你的用户名/docflow.git`

---

## 第二步：上传代码到GitHub

### 方法A：使用GitHub网页（最简单）

1. 在仓库页面点击 **uploading an existing file**
2. 打开 `C:\Users\Administrator\WorkBuddy\20260406003053\docflow` 文件夹
3. 全选所有文件和文件夹上传
4. 点击 **Commit changes**

### 方法B：使用Git命令

```bash
cd C:\Users\Administrator\WorkBuddy\20260406003053\docflow
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/docflow.git
git push -u origin main
```

---

## 第三步：部署到Vercel

### 操作
1. 打开 https://vercel.com
2. 点击 **Sign Up**（用GitHub登录）
3. 授权GitHub访问
4. 点击 **Import Project**
5. 选择你刚创建的 `docflow` 仓库
6. 点击 **Deploy**

### 等待
部署需要3-5分钟，Vercel会自动构建。

### 完成
部署成功后，你会看到一个URL，类似：`https://docflow-xxx.vercel.app`

---

## 第四步：测试API

### 操作
在浏览器或终端测试：

```bash
curl -X POST https://docflow-xxx.vercel.app/api/v1/generate \
  -H "Content-Type: application/json" \
  -H "x-api-key: docflow_test_key_12345678901234567890" \
  -d '{
    "template": "invoice",
    "data": {
      "company": "测试公司",
      "items": [{"name": "测试产品", "amount": 1000}],
      "total": 1000
    }
  }'
```

如果返回PDF文件，说明部署成功！

---

## 第五步：创建正式API密钥（可选）

### 操作
1. 打开你的网站首页（Vercel给的URL）
2. 输入你的邮箱
3. 点击"获取API密钥"
4. 保存生成的密钥

---

## 第六步：配置Stripe支付（可选）

如果你想收费功能：

1. 注册 Stripe: https://stripe.com
2. 在Vercel环境变量中添加：
   - `STRIPE_SECRET_KEY`
   - `STRIPE_WEBHOOK_SECRET`
3. 实现支付回调逻辑

## 常见问题

### Q: 部署失败了怎么办？
A: 检查错误信息，常见问题：
- Node.js版本不兼容（需要18+）
- 依赖安装失败

### Q: API返回500错误？
A: 开发模式下PDF生成需要Chrome，可能在Vercel上需要额外配置

### Q: 怎么绑定自己的域名？
A: 在Vercel项目设置 → Domains 添加域名
